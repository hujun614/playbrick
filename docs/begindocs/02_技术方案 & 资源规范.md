# 《像素砖块冲击》技术方案 & 资源规范文档
## 1 基础技术栈
- 引擎：Phaser 3.80
- 构建工具：Vite
- 语言：TypeScript
- 目标平台：抖音小游戏
- 屏幕分辨率：竖屏 720 × 1280
- 物理引擎：Arcade Physics（轻量2D，适配移动端）

## 3 资源强制规范（核心解决空贴图、资源丢失报错）
3.1 资源命名规则
- 图片：小写蛇形命名 paddle_normal、brick_gold、item_double_score
- 音频：hit_brick、get_item、lose_life
- 禁止中文、空格、特殊符号
3.2 资源索引规则
所有资源Key统一写在 `resourceKey.ts`，全局仅通过常量调用，禁止手写字符串路径
示例：
```ts
export const RES_KEY = {
  PADDLE_NORMAL: "paddle_normal",
  BRICK_GOLD: "brick_gold"
}

3.3 加载规则
全部资源在 Boot 场景一次性预加载完成
ResourceManager 统一捕获加载失败，控制台打印缺失资源 Key，直接定位问题
游戏运行阶段不动态加载资源，避免中途贴图空引用
4 核心技术实现方案
4.1 物理碰撞分层管理
分 4 个独立物理分组，仅配置需要的碰撞检测，减少性能消耗
bricksGroup 静态砖块（immovable:true）
ballGroup 动态小球
itemGroup 掉落道具
paddle 玩家挡板
碰撞逻辑全部代码绑定，无需编辑器勾选碰撞体
4.2 对象池复用机制
小球、掉落道具使用对象池：
超出屏幕 / 销毁时回收至池，不 destroy
需要生成时从池中取出空闲实例，重置位置与速度
优势：减少 GC 卡顿，避免频繁 new 导致的性能波动
4.3 触屏输入封装
统一输入管理类，区分 PC 键盘 / 手机触摸，对外提供标准化挡板移动接口
4.4 抖音小游戏适配
本地存储封装：封装 setScore、getMaxScore，兼容抖音 tt.getStorageSync
分包规则：资源单独分包，减少主包体积
生命周期适配：小游戏前后台切换暂停 / 恢复游戏
隐私弹窗、基础提审配置预留代码位置
5 代码规范
所有游戏数值统一读取配置文件，禁止硬编码数字
精灵创建全部代码 new，无外部场景文件、无拖拽节点
每个实体（挡板 / 球 / 砖块）单独抽类，解耦业务逻辑
注释规范：功能函数、关键物理参数、资源 Key 添加注释
